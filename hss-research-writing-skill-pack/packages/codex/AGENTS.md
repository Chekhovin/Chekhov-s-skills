# HSS research writing instructions for Codex

When working on humanities or social-science research in this repository, default to a *materials-first* workflow.

## Purpose

Turn raw materials into defensible arguments before turning arguments into prose.

## Non-negotiables

- Never fabricate citations, page numbers, archive identifiers, case numbers, interview metadata, or numerical values.
- Distinguish source report, your interpretation, your inference, and unresolved hypothesis.
- Every important claim should map to concrete evidence.
- If a fact cannot be verified, keep a visible placeholder such as `[待核]`.
- Do not write generic “importance” language without saying what is new.
- Preserve ethics and anonymization for interviews, fieldnotes, and sensitive cases.

## Default workflow

1. Create or update `research-workbench/00-task-brief.md`.
2. Build `research-workbench/10-source-ledger.md`.
3. Extract source cards / matrices in `research-workbench/20-reading-memos/` or `30-coding/`.
4. Build `research-workbench/40-evidence-matrix.md` and `50-argument-map.md`.
5. Only then draft or revise prose in `60-outline.md` and `70-draft.md`.
6. Track changes in `80-revision-log.md`.

If the user asks for a fast draft before analysis, still perform a lightweight evidence check and clearly mark weak or pending sections.

## Writing rules

- One paragraph should do one analytic job.
- A literature review should map disputes, not list authors.
- A quote must do analytic work, not decorative work.
- Use precise, concrete verbs. Cut filler and automatic transitions.
- Keep the prose readable aloud; if a sentence cannot be said naturally to a sharp colleague, revise it.

## Final checks

Before treating anything as near-final, audit:

- unsupported claims
- unverified quotes or page numbers
- concept drift
- missing counterevidence
- AI-sounding filler or inflated significance language

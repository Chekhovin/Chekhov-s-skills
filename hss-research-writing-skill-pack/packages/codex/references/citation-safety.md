# Citation safety and source integrity

这是本技能里最不能放松的一部分。

## 1. 总原则

- 不从记忆里“补齐”学术引文。
- 不编作者名、年份、页码、卷期、DOI、档案号、案号。
- 不把二手转引伪装成自己读过原文。
- 不把意译伪装成直接引语。
- 不把未核信息写成定稿格式。

## 2. 直接引语

### 必须满足

- 原文措辞准确
- 标点和删节处理清楚
- 页码 / 条号 / 段号尽量准确
- 若为译引，注明“作者译”或同义提示

### 如果拿不准

使用这种占位方式：

- `[引文原文待核]`
- `[页码待核]`
- `[译文待核]`

## 3. 间接引述

间接引述也不能“想当然”。需要确认：

- 这个判断确实是该作者主张的，不是你替他延伸的
- 不要把作者的研究对象扩大成普遍命题
- 不要把语境限定很强的话改写成普遍结论

## 4. 二手转引

如果你没看原文，但在别人的文献中看到：

- 最稳妥做法是说明“转引自”
- 不要假装自己直接读了原文
- 正式投稿前尽量回到原始来源

## 5. 档案 / 一手材料

尽量记录：

- 馆藏 / 全宗 / 卷宗 / 盒号 / 文件号
- 文种
- 日期
- 生产者
- 页码 / 页序
- 访问方式（实体 / 数字库 / 复印件）

如果不全，显式写：`[档号待核]`，不要空想一个“像真的”编号。

## 6. 访谈 / 田野

如果是匿名材料：

- 用一致的化名体系
- 不暴露可回溯身份的细节组合
- 日期、地点、职业标签可以必要模糊

同时记录内部研究台账，不一定全部进正文：

- 访谈编号
- 日期
- 场景
- 访谈者 / 受访者角色
- 是否录音 / 是否转写

## 7. 统计数据 / 表格 / 数值

### 必须避免

- 凭印象写一个“大概正确”的百分比
- 把图里看出来的趋势写成精确数值
- 引用次级报道中的数字，却装作来自原始数据库

### 更稳的处理方式

- 数值未核时写成描述性表达：`“明显上升”`、`“约为”`、`“具体数值待核”`
- 或者直接保留占位：`[统计值待核]`

## 8. 风格未知时怎么办

如果用户没指定 Chicago / APA / MLA / GB/T 7714：

- 不要强行排成看起来正式但实则不准确的参考文献表
- 优先先把信息整理齐，再决定格式
- 可以先用中性信息块：

```text
Author / Institution:
Year / Date:
Title:
Container / Publisher:
Volume(Issue):
Pages:
DOI / URL:
Status: verified / partial / pending
```

## 9. 待核标记要保留到最后一轮清理

推荐统一使用：

- `[待核]`
- `[页码待核]`
- `[引文原文待核]`
- `[档号待核]`
- `[统计值待核]`

不要在中途为了“看起来整洁”把这些删掉。

## 10. 提交前最后清单

投稿前至少逐项检查：

- 每条直接引语都有对应来源与页码
- 每条参考文献都真实存在
- 作者、年份、题名、期刊 / 出版社准确
- 档案与法律材料编号真实可回溯
- 访谈匿名规则全篇一致
- 所有 `[待核]` 项都被处理或被显式保留说明

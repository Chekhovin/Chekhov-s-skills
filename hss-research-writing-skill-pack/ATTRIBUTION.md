# Attribution and design notes

这个技能包是一个*原创综合改写*，不是对任一上游仓库的逐字复制。

它主要借鉴了以下公开项目的思路，并针对人文社会科学研究场景做了重构：

1. `Orchestra-Research/AI-Research-SKILLs/20-ml-paper-writing`
   - 借鉴点：从项目/材料出发的主动式写作流程、贡献句、引用核验、先做结构再做初稿。
   - 本包改造：把“实验仓库 -> 论文”的逻辑改成“材料台账 -> 证据矩阵 -> 论证地图 -> 论文/综述/研究计划”的 HSS 工作流。

2. `blader/humanizer`
   - 借鉴点：识别和清理 AI 写作痕迹、做最终的反模板审计。
   - 本包改造：保留“反 AI 腔”思路，但改成适合学术写作的低噪版本，避免把论文改成过度口语化文本。

3. `ginobefun/deep-reading-analyst-skill`
   - 借鉴点：分层分析、思维框架、从材料理解到综合输出的五步流程。
   - 本包改造：把通用深阅读框架变成文献综述、档案、访谈、政策/法律/媒体、理论研究等 HSS 专用分析路线。

4. `lijigang/ljg-skills/skills/ljg-writes`
   - 借鉴点：口语检验、句子密度、反模板、用具体场景而不是空话推进论证。
   - 本包改造：保留“句子要经得起说出口”的要求，但把语体控制在学术论文和研究备忘录能接受的范围内。

5. `QJHWC/PaperForge`
   - 借鉴点：阶段化流程、任务路由、直接执行而不是只返回 prompt、多轮修订与自检。
   - 本包改造：把“实验-优化-写作”流程改造成“采集-提炼-比较-综合-写作-审计”的人文社科版流程，并增加 AGENTS / Agent Skills / CONVENTIONS 适配层。

## 额外说明

- 本包偏重*工作流设计*、*提示规范*、*模板结构*和*跨工具适配*。
- 如果你用于正式发表，请自行复核所有引用、页码、档案信息、引语、统计值与伦理处理。
- 若要对外二次分发，建议你自行检查各上游仓库的最新许可条款。
